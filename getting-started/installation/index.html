Installation

Before you begin

  • You need macOS (arm64 or x64), Linux (x64), or Windows (arm64 or x64).
  • You need an Atlassian account with access to the products you want to use.
  • You need an AI coding agent installed on your computer to use TWG agent skills. This guide uses Claude Code as an example. For other supported agents, see What is TWG CLI?.

The TWG CLI binary does not require a separate runtime (for example, no Node.js).

Install an AI coding agent

TWG CLI works with any AI coding agent. This guide uses Claude Code as an example.

If you already have an AI coding agent installed, skip this section and continue with macOS and Linux or Windows.

Install Claude Code

Note

The Homebrew and shell script options below are for macOS and Linux. On Windows, follow the install steps for Claude Code (or your chosen agent) from the vendor's documentation.

  1. Open Terminal.

  2. Run:

    brew install claude-code

    This requires Homebrew. If you do not have Homebrew, you can install Claude Code with:

    curl -fsSL https://claude.ai/install.sh | bash

  3. Follow the prompts to complete the installation.

macOS and Linux

Using an AI coding agent? Give it only the hosted install instructions: https://teamwork-graph.atlassian.com/cli/AGENTS.md.

Run the install script:

curl -fsSL https://teamwork-graph.atlassian.com/cli/install | bash

The installer detects your platform, verifies the download checksum, installs the binary in ~/.local/bin, asks for first-time consent, installs TWG agent skills, refreshes command help, and starts authentication.

When consent is required, answer yes, continue, or no. The installer then opens the token page selected by the binary. In a direct terminal, generate a token, copy it in the browser, then press Enter; the installer reads it from the OS clipboard and falls back to a masked paste prompt only if clipboard access is unavailable. If a Bitbucket token is not already configured, setup offers the same clipboard-confirmed flow for the optional Bitbucket token. If you use clipboard history, clear it after setup. If the binary is using classic API-token mode, the installer also asks for your Atlassian account email because classic tokens cannot identify the account by themselves. Agent sessions should follow the hosted instructions above for the consent and token handoff.

The installer completes setup automatically. To repair setup later:

twg setup

To install a specific version:

curl -fsSL https://teamwork-graph.atlassian.com/cli/install | bash -s -- --version 0.9.1

If ~/.local/bin is not in your PATH, the installer prints the exact line to add to your shell profile.

Note

When prompted for credentials, see Authentication for instructions on creating an API token.

Homebrew (macOS)

brew tap atlassian/twg
brew install twg
twg setup

Windows

Install — PowerShell

irm https://teamwork-graph.atlassian.com/cli/install.ps1 | iex

Install — Command Prompt (cmd)

Note

curl is built into Windows 10 1803 and later. If you're on an older version, use the PowerShell method above instead.

powershell -ExecutionPolicy Bypass -Command "irm https://teamwork-graph.atlassian.com/cli/install.ps1 | iex"

The Windows installer performs the same setup as macOS and Linux: binary install, checksum verification, skill install, help refresh, and token login.

Verify the installation

twg doctor

This works the same on macOS, Linux, PowerShell, and cmd.

Update

macOS/Linux — use the built-in updater:

twg update

macOS (Homebrew):

brew update && brew upgrade twg

Windows (PowerShell or cmd) — re-run the install command from the Windows section above.

twg update and the installers refresh installed skill files automatically when they can detect existing TWG skill installs.
