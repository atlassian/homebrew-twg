Authentication

Log in

Run the following command and follow the prompts:

twg login

You'll be asked for:

Prompt What to enter
Email Your Atlassian account email
TWG token A scoped API token — twg login opens your browser automatically
Bitbucket token Optional separate token for Bitbucket-specific commands
Site Your Atlassian site name (for example, example for example.atlassian.net)

Credentials are saved to ~/.twg/auth.conf and reused across sessions.

Create a TWG API token

TWG CLI uses a single scoped API token for TWG commands across Jira, Confluence, Rovo Search, and more.

Bitbucket-specific commands use Bitbucket Cloud APIs and require a separate Bitbucket token. twg login prompts for this optional token, and preserves an existing Bitbucket token unless you run twg login --force.

twg login opens your browser automatically to the token creation page. If you need to create one manually:

  1. Go to https://id.atlassian.com/manage-profile/security/api-tokens?autofillToken=&appId=twg&selectedScopes=all&expiryDays=30
  2. Click Create API token with scopes and accept the pre-filled settings.
  3. Copy the token value and paste it when twg login prompts you.

Use environment variables

For CI/CD and scripted use, set the following variables instead of running twg login:

Variable Purpose
TWG_USER Atlassian account email
TWG_SITE Default site prefix
TWG_TOKEN TWG scoped API token
TWG_BBC_TOKEN Bitbucket token for Bitbucket-specific commands
Note

Never pass tokens as CLI flags. TWG CLI only accepts credentials interactively or via environment variables.
